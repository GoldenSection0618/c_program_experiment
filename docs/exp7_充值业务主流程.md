# 实验七：充值业务主流程

## 1. 目标
本阶段只实现菜单 `5. 充值`，建立从用户输入到卡余额更新、充值记录落盘和结果展示的完整闭环。

## 2. 输入项
充值流程固定读取三项输入：
- 卡号
- 密码
- 充值金额（元）

## 3. 充值前置条件
满足以下条件才允许充值：
- 卡存在
- 密码正确
- 卡未被逻辑删除
- 卡状态为 `CARD_STATUS_OFFLINE` 或 `CARD_STATUS_ONLINE`
- 卡状态不是 `CARD_STATUS_CANCELED`
- 充值金额为有效正数
- 充值后余额仍小于 `MAX_BALANCE_CENT`

## 4. 充值记录写入规则
充值成功时会追加一条 `Money` 记录到 `data/money.txt`：
- `aCardName`：卡号
- `tTime`：充值时间
- `nStatus = 0`：表示充值
- `nMoneyCent`：充值金额，单位为分
- `nDel = 0`

## 5. 双写顺序
当前充值主流程采用以下顺序：
1. 先更新卡余额并调用 `dataUpdateCard()`
2. 再调用 `dataSaveMoney()` 追加充值记录

若第二步失败，则立即用原卡快照回滚 `dataUpdateCard()`，避免出现“余额已变化但无充值记录”的脏状态。

## 6. 展示结果
充值成功后，控制台展示：
- 卡号
- 充值金额
- 当前余额

充值失败时，先输出 `充值失败！`，再输出对应错误提示。
