# 实验一：计费管理系统需求分析与菜单界面设计

## 1. 实验目标
本实验用于搭建计费管理系统的整体框架，完成控制台主菜单与各业务模块入口设计，建立后续实验可持续填充的代码骨架。

## 2. 范围说明
### 2.1 本实验实现范围
1. 设计并实现主菜单界面与循环交互流程。
2. 建立卡管理、计费标准管理、计费管理、费用管理、查询统计五大模块的函数入口。
3. 建立基础数据结构和分层组织方式（表示层、业务逻辑层、数据存储层）。
4. 对菜单输入进行合法性校验，保证程序稳定运行。

### 2.2 本实验不实现内容
1. 不要求一次性完成所有业务模块的完整实现。
2. 计费算法、统计计算、数据持久化等可在后续实验逐步完善。

## 3. 功能需求分析
说明：本章保留系统完整业务需求细节，作为后续实验逐步实现的目标基线。实验一需先完成菜单入口与框架对接。

### 3.1 卡管理
1. 新增卡
- 输入：卡号、密码、开卡金额。
- 处理：校验卡号唯一性、密码合法性、初始金额非负。
- 输出：开卡成功/失败原因。

2. 查询卡
- 输入：卡号。
- 处理：按卡号检索卡信息。
- 输出：卡号、卡状态、余额、累计使用金额、使用次数、最后使用时间等。

3. 注销卡
- 输入：卡号、必要身份校验信息。
- 处理：仅允许未注销卡执行注销，注销后冻结全部业务操作。
- 输出：注销成功/失败原因。

### 3.2 计费标准管理
1. 新增标准
- 输入：标准编号、单价、计费单位、生效时间等。
- 输出：新增结果。

2. 查询标准
- 输入：可选筛选条件（全部/按编号）。
- 输出：标准列表。

3. 删除标准
- 输入：标准编号。
- 处理：校验标准是否存在、是否允许删除。
- 输出：删除结果。

4. 修改标准
- 输入：标准编号、新标准参数。
- 输出：修改结果。

### 3.3 计费管理
1. 上机
- 输入：卡号、密码。
- 处理：校验卡状态、密码、余额是否满足最小上机条件，生成上机记录并开始计费。
- 输出：上机成功/失败原因、开始时间。

2. 下机
- 输入：卡号、密码。
- 处理：校验是否处于上机状态，按计费标准计算消费金额并扣费。
- 输出：下机成功/失败原因、上机时长、消费金额、余额。

### 3.4 费用管理
1. 充值
- 输入：卡号、充值金额。
- 处理：金额合法性校验并累加余额。
- 输出：充值结果、充值后余额。

2. 退费
- 输入：卡号、身份校验信息。
- 处理：校验不可上机状态后按余额退费并清零/关闭。
- 输出：退费金额、退费后状态。

### 3.5 查询统计
1. 消费记录查询
- 输入：卡号、起止时间。
- 输出：该卡在时间段内的消费明细。

2. 统计总营业额
- 输入：起止时间。
- 输出：时间段总营业额。

3. 统计月营业额
- 输入：年份。
- 输出：该年 1~12 月营业额统计结果。

## 4. 非功能与约束
1. 可用性：控制台菜单需稳定循环，非法输入不崩溃。
2. 可维护性：采用表示层、业务逻辑层、数据存储层分层结构。
3. 可扩展性：菜单入口与业务函数解耦，便于后续实现具体业务。
4. 输入健壮性：对 `1w`、`ttt` 等非纯数字输入需提示错误并继续交互。

## 5. 人机接口（控制台主界面）需求映射
1. 主菜单首行显示：学号-姓名-班级。
2. 显示系统名称：计费管理系统。
3. 菜单项编号：1~8 对应功能，0 退出。
4. 采用 `switch-case` 显示用户所选功能名称。
5. 采用 `do...while` 循环，输入 0 时退出。

## 6. 输入输出示例（主菜单）
- 输入：`1` -> 输出：你选择了：添加卡
- 输入：`9` -> 输出：无效菜单编号，请输入 0~8
- 输入：`1w` 或 `ttt` -> 输出：输入格式错误，请输入数字菜单编号（0~8）
- 输入：`0` -> 输出：系统已退出

## 7. 实验一验收标准
1. 程序启动后可正确显示主菜单信息。
2. 输入 `1~8` 时可进入对应模块入口并返回菜单。
3. 输入非法编号（如 `9`）时提示“无效菜单编号”。
4. 输入非法格式（如 `1w`、`ttt`）时提示“输入格式错误”。
5. 输入 `0` 时程序正常退出。
6. 保留第 3 章细化需求作为后续实验实现基线。
