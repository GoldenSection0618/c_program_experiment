# 实验七：退费业务主流程

## 1. 目标
本阶段只实现菜单 `6. 退费`，建立从用户输入到卡余额清零、退费记录落盘和结果展示的完整闭环。

## 2. 输入项
退费流程固定读取两项输入：
- 卡号
- 密码

## 3. 退费前置条件
满足以下条件才允许退费：
- 卡存在
- 密码正确
- 卡未被逻辑删除
- 卡状态必须为 `CARD_STATUS_OFFLINE`
- 已注销卡不允许退费
- 当前余额必须大于 `0`

## 4. 退费金额规则
- 退费金额固定等于当前卡余额
- 不支持部分退费
- 退费成功后卡余额变为 `0`

## 5. 退费记录写入规则
退费成功时会追加一条 `Money` 记录到 `data/money.txt`：
- `aCardName`：卡号
- `tTime`：退费时间
- `nStatus = 1`：表示退费
- `nMoneyCent`：退费金额，单位为分
- `nDel = 0`

## 6. 双写顺序
当前退费主流程采用以下顺序：
1. 先更新卡余额并调用 `dataUpdateCard()`
2. 再调用 `dataSaveMoney()` 追加退费记录

若第二步失败，则立即用原卡快照回滚 `dataUpdateCard()`，避免出现“余额已清零但无退费记录”的脏状态。

## 7. 展示结果
退费成功后，控制台展示：
- 卡号
- 退费金额
- 当前余额

退费失败时，先输出 `退费失败！`，再输出对应错误提示。
