# 实验六：输出格式和用户体验整理

## 1. 本轮目标
实验六迭代 5 不再修改上机、下机的核心算法，只整理控制台输出格式和失败提示，使菜单 3、4 的交互更一致。

## 2. 成功输出整理
### 2.1 上机成功
上机成功后继续输出表格，至少展示三列：
- 卡号
- 余额
- 上机时间

### 2.2 下机成功
下机成功后输出结算结果表格，固定展示：
- 卡号
- 上机时间
- 下机时间
- 消费金额
- 结算后余额

其中“结算后余额”用于明确区分结算前的卡内余额和下机扣费后的余额。

## 3. 失败提示统一
### 3.1 上机失败
上机失败时统一先输出：
- `上机失败！`

随后根据原因输出细化提示，例如：
- `卡号或密码错误！`
- `该卡正在使用！`
- `该卡已注销！`
- `余额不足！`
- 文件异常类提示

### 3.2 下机失败
下机失败时统一先输出：
- `下机失败！`

随后根据原因输出细化提示，例如：
- `卡号或密码错误！`
- `未找到该卡的未结算消费记录！`
- 文件异常类提示

## 4. 本轮结论
实验六迭代 5 完成后：
- 菜单 3、4 的成功输出字段已经统一
- 上机、下机失败时都有统一前缀
- 卡号不存在与密码错误场景统一收敛为同一条用户提示
- 下机不再因余额不足失败；若形成欠费，按成功结果展示结算后余额
- 交互体验更一致，但业务判断和持久化逻辑保持不变
