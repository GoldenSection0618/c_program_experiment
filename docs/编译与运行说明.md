# 计费管理系统编译与运行说明

## 1. 环境要求
- 操作系统：Linux / macOS / Windows（MinGW）
- 编译器：`gcc`（支持 C11）
- 构建工具：`make`
- 可选前端工具（按优先级自动检测）：`pnpm` / `npm` / `yarn`

## 2. C 程序编译
默认 `make` 使用 `release` 模式编译：
```bash
make
```
- 产物路径：`build/bin/billing_system`

指定模式编译：
```bash
make MODE=debug
make MODE=release
```

或使用快捷目标：
```bash
make debug
make release
```

## 3. 运行
推荐：
```bash
make run
```

等价命令：
```bash
./build/bin/billing_system
```

## 4. 烟测（内置验收）
```bash
make test
```
- `Makefile` 不再硬编码测试用例，测试由文件驱动：
  - 默认输入文件：`tests/exp2/menu.input`
  - 默认断言文件：`tests/exp2/menu.expect.tsv`
  - 执行脚本：`scripts/run_smoke_test.sh`
- 输出日志：`build/test_output.txt`

切换实验/套件示例（后续实验可直接扩展 `tests/` 目录）：
```bash
make test LAB=exp2 TEST_SUITE=menu
```

实验二已补充回归测试套件（用于边界问题验证）：
```bash
make test LAB=exp2 TEST_SUITE=menu_regression_long_input
make test LAB=exp2 TEST_SUITE=menu_regression_whitespace
```

批量执行实验二套件：
```bash
make test-exp2-acceptance
make test-exp2-regression
```

## 5. 前端目标（占位可执行）
若项目中不存在 `frontend/` 目录，以下命令会提示 `skip` 并安全返回：
```bash
make frontend-install
make frontend-build
make frontend-dev
```

若存在 `frontend/`，将按 `pnpm -> npm -> yarn` 优先级自动选择工具执行。

## 6. 清理命令
```bash
make clean
```
- 删除 `build/` 目录（含 `obj/dep/bin` 及测试输出）

```bash
make distclean
```
- 在 `make clean` 基础上，额外尝试删除 `frontend/node_modules`

## 7. 常用命令速查
```bash
make help
```
