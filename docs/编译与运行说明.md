# 计费管理系统编译与运行说明

## 1. 环境要求
- 操作系统：Linux / macOS / Windows（MinGW）
- 编译器：`gcc`（支持 C11）
- 构建工具：`make`
- 可选前端工具（按优先级自动检测）：`pnpm` / `npm` / `yarn`

## 2. C 程序编译
默认 `make` 使用 `release` 模式编译：
```bash
make
```
- 产物路径：`build/bin/billing_system`

指定模式编译：
```bash
make MODE=debug
make MODE=release
```

或使用快捷目标：
```bash
make debug
make release
```

## 3. 运行
推荐：
```bash
make run
```

等价命令：
```bash
./build/bin/billing_system
```

## 4. 测试
默认测试已切换到实验三主流程：
```bash
make test
```
- 默认输入文件：`tests/exp3/menu.input`
- 默认断言文件：`tests/exp3/menu.expect.tsv`
- 输出日志：`build/test_output.txt`

批量执行实验三测试：
```bash
make test-batch LAB=exp3 TEST_GROUP=acceptance
make test-batch LAB=exp3 TEST_GROUP=regression
```

执行实验四链表删除测试：
```bash
make test LAB=exp4 TEST_SUITE=repository_delete_api
make test-batch LAB=exp4 TEST_GROUP=acceptance
```

如需切换历史实验：
```bash
make test LAB=exp1 TEST_SUITE=menu
make test LAB=exp2 TEST_SUITE=menu
make test-batch LAB=exp1 TEST_GROUP=acceptance
make test-batch LAB=exp2 TEST_GROUP=acceptance
```

说明：
- 测试仍由文件驱动，不在 `Makefile` 中硬编码测试用例
- `scripts/run_smoke_test.sh` 已支持套件级 `*.setup.sh` / `*.verify.sh` 钩子
- 实验三运行过程中会使用相对路径 `data/card.bin` 保存卡数据

## 5. 前端目标（占位可执行）
若项目中不存在 `frontend/` 目录，以下命令会提示 `skip` 并安全返回：
```bash
make frontend-install
make frontend-build
make frontend-dev
```

若存在 `frontend/`，将按 `pnpm -> npm -> yarn` 优先级自动选择工具执行。

## 6. 清理命令
```bash
make clean
```
- 删除 `build/` 目录（含 `obj/dep/bin` 及测试输出）

```bash
make distclean
```
- 在 `make clean` 基础上，额外尝试删除 `frontend/node_modules`

## 7. 常用命令速查
```bash
make help
```
