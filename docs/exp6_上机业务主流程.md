# 实验六：上机业务主流程

## 1. 本轮目标
实验六迭代 2 只实现“上机”，不进入下机结算。

本轮完成后，菜单 `3. 上机` 需要形成完整闭环：
- 输入卡号和密码
- 从卡文件恢复卡链表
- 从消费记录文件恢复消费记录链表
- 校验是否允许上机
- 更新卡状态为“正在上机”
- 追加一条未结算消费记录
- 输出上机结果

## 2. 上机前置条件
上机前必须满足：
- 卡存在
- 密码正确
- 卡未删除
- 卡状态为 `CARD_STATUS_OFFLINE`
- 余额满足最低要求：`nBalanceCent >= 0`

若任一条件不满足，直接失败，不修改卡文件和消费记录文件。

## 3. 上机成功后的动作
1. 记录当前时间为上机时间
2. 更新卡状态为 `CARD_STATUS_ONLINE`
3. 将更新后的卡信息写回 `data/cards.txt`
4. 追加一条新的未结算消费记录到 `data/billings.txt`
   - 卡号 = 当前卡号
   - 上机时间 = 当前时间
   - 下机时间 = `0`
   - 消费金额 = `0`
   - 消费状态 = `0`（未结算）
   - 删除标识 = `0`
5. 输出上机结果：
   - 卡号
   - 余额
   - 上机时间

## 4. 一致性处理
本轮最关键的保护点是：
- 若卡状态已成功写回，但追加消费记录失败，必须立即将卡信息回滚到原状态

因此本轮避免了这类脏状态：
- 卡已经变成“正在上机”，但没有对应未结算消费记录

## 5. 本轮结论
实验六迭代 2 完成后，项目已经具备：
- 菜单 3 的真实交互
- 上机条件校验
- 成功上机时的卡状态更新
- 成功上机时的未结算消费记录追加
- 失败场景下的不半更新保护

下一轮可以继续进入：
1. 下机业务主流程
2. 根据未结算消费记录完成结算
3. 同步更新卡信息与消费记录
